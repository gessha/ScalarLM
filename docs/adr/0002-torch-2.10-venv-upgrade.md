# Upgrade torch to 2.10 inside the venv, not in the base image

vllm-fork requires torch == 2.10.0: it uses the `TORCH_BOX` macro introduced in torch 2.10 (`csrc/libtorch_stable/torch_bindings.cpp`), and its `requirements/cuda.txt` pins `torch==2.10.0`. NGC 25.10 ships torch 2.9.

We install `torch==2.10.0+cu128` (and matching torchvision, torchaudio) via pip into the system venv. The venv uses `--system-site-packages`, so the installed version takes precedence over the NGC system torch.

`--force-reinstall` is required: without it, pip finds the NGC system torch and prints "Requirement already satisfied" without installing anything into the venv.

## Considered options

- **Patch vllm-fork to work with torch 2.9** — not viable; `TORCH_BOX` is a hard compile-time dependency.
- **Use NGC 26.01 (torch 2.10 natively)** — blocked by driver constraint; see ADR-0001.
- **Upgrade torch in the venv** — accepted. The venv shadows NGC system packages.

## Consequences

The NGC system flash_attn, xformers, and other packages compiled against torch 2.9 become ABI-incompatible and must be shadowed. See ADR-0003.
